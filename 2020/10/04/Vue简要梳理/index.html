Vue基本语法

  • v-开头为指令：

    • v-bind:我们可以使用v-bind来绑定元素属性，当元素属性值为model层data数据中的变量时，元素属性和Vue中的data数据就可以实现动态变化。

    • v-if/v-else:在元素标签中间增加上述关键字，在之后用等号相连，并在等号后面双引号写入判断条件语句，当判断条件结果为true时，元素标签得以显示。

    • v-for:后面同样跟等号，等号中为 “元素 in 元素集合” 的格式，元素集合为Vue实例中data中的变量集合。

    • v-on:类似于将JavaScript中的操作关键字之前加上上述关键字，事件相关的函数在Vue实例的methods中定义。

    • v-model:可以实现数据的双向绑定。当数据

  • Vue中常用的七大属性

    • el属性
      绑定需要解析为Vue语法的元素位置，如 el:”#app”。
    • data属性
      页面需要用到的数据都放在这个地方。
    • template属性
      模板属性中写自定义的标签名代表的HTML元素标签的组合。要使得自己定义的标签能够使用，需要先注册组件，即先完成Vue.Component()部分，再实例化Vue，即再完成var vm=new Vue()。
    • methods属性
      在这个属性中自定义函数并完成相应逻辑。
    • render属性
    • computed属性
      在这个属性中可以写入和methods中一样的函数逻辑，只是在div中调用时，计算属性中的值可以当作属性直接调用，而在函数中定义的方法需要作为方法调用。
    • watch侦听属性

创建vue项目步骤

  • 确保本地安装了vue-cli，类似于要使用Maven管理项目时需要本地有Maven服务。vue-cli需要的环境：Node.js、git。

  • 创建一个基于webpack模板的vue应用程序：vue init webpack hello-vue (其中的hello-vue为项目名)(一路选择no，最后选择“自己运行npm install”)

  • 安装依赖， 我们需要安装vue-router、element-ui、sass-loader和node-sass四个插件：

  • 完美运行，则项目创建成功。

  • 项目常用目录结构（src目录下）

    • assets：用于存放资源文件
    • components：用于存放Vue功能组件
    • views：用于存放Vue视图组件
    • router：用于存放vue-router配置

  • 先创建vue组件，之后在路由配置文件中使用import关键字导入，在routes属性中配置相关路由path和希望对应的组件component即可。

异步请求

  • Vue较早使用的异步请求工具为vue-resource，vue-resource是vue官方团队的人开发的，在vue2.0出来之后，开发vue-resource的人表示不再更新，放弃维护，并推荐使用axios
  • 向服务器发送请求需要安装Axios ，命令为cnpm install --save vue-axios
  • 之后需要在main.js引用 Axios

Tips

  • 钩子函数：类似于小程序中的onLoad、onShow等函数，在特定动作完成之前可以在钩子函数中进行预处理。

  • 路由模式有两种

    • hash：路径带 # 符号，如 http://localhost/#/login
      默认为hash路由模式
    • history：路径不带 # 符号，如 http://localhost/login
      history路由模式

  • 实现404页面只需将所有未知的路由，即path: '*'的情况都跳转到我们设置的404页面即可

  • 在组件中的data要写成方法，其中的数据由return返回
